### 1. Node.js 개념
Node.js는 Chrome의 V8 엔진 위에서 동작하는 JavaScript 런타임 환경으로, 브라우저 밖에서도 JavaScript를 실행할 수 있도록 한다.  
V8 엔진이 JavaScript 코드를 기계어로 변환해 실행하고, Node.js는 파일 시스템, 네트워크, 프로세스 제어 기능을 제공하여 서버 애플리케이션 개발이 가능하도록 확장한다.

---

### 2. 비동기 처리와 이벤트 루프
Node.js의 핵심 특징은 이벤트 루프 기반의 비동기 처리 구조이다.  
데이터베이스 조회나 파일 I/O와 같은 시간이 오래 걸리는 작업은 백그라운드에서 처리하고, 완료 시 콜백, Promise, async/await 형태로 이벤트 큐에 등록된다.  

이벤트 루프는 이를 순차적으로 실행하여 단일 스레드 구조에서도 높은 처리 효율을 유지하며, 특히 I/O 중심 서비스에서 성능상의 이점을 가진다.

---

### 3. 서버 동작 구조
Node.js 서버는 클라이언트의 HTTP 요청을 받아 Express와 같은 프레임워크를 통해 라우팅을 수행하고,  
미들웨어에서 인증, 세션, 요청 데이터 처리 등의 공통 로직을 실행한 뒤, 컨트롤러에서 비즈니스 로직을 처리한다.  

이후 데이터베이스와 비동기 통신을 거쳐 결과를 JSON 형태로 응답하며 요청-응답 사이클이 완료된다.

---

### 4. 프로젝트 적용
본 프로젝트에서는 Node.js와 Express를 기반으로 RESTful API 서버를 구축하여 게시글 CRUD, 사용자 인증, 댓글 및 투표 기능을 구현하였다.  

MongoDB와의 연동은 async/await를 활용해 비동기적으로 처리했으며, Socket.IO를 통해 실시간 채팅 기능을 구현하여 Node.js의 이벤트 기반 처리 구조를 실제 서비스 기능에 적용하였다.
### Realtime
- WebSocket (Socket.io): 실시간 채팅 및 상태 동기화 구현

### 1. WebSocket 개념
WebSocket은 클라이언트와 서버 간의 양방향 실시간 통신을 지원하는 프로토콜이다.  
기존 HTTP 통신은 요청(Request)과 응답(Response)으로 연결이 종료되는 반면, WebSocket은 한 번 연결이 수립되면 연결을 유지한 상태에서 데이터를 주고받을 수 있다.

---

### 2. 동작 방식
WebSocket은 초기에는 HTTP 핸드셰이크 과정을 통해 연결을 수립한 후, 프로토콜을 WebSocket으로 업그레이드한다.  
이후에는 클라이언트와 서버 간에 지속적인 연결(Persistent Connection)을 유지하며, 양방향으로 데이터를 송수신할 수 있다.  

이 방식은 서버가 클라이언트의 요청을 기다리지 않고도 데이터를 즉시 전송할 수 있기 때문에 실시간성이 중요한 서비스에 적합하다.

---

### 3. WebSocket의 필요성
HTTP 기반 통신에서는 실시간 데이터를 처리하기 위해 polling 방식이나 long polling을 사용해야 했는데, 이는 불필요한 요청이 반복되어 비효율적이다.  

WebSocket은 연결을 유지하면서 필요한 시점에만 데이터를 주고받기 때문에 네트워크 오버헤드를 줄이고, 빠른 응답 속도를 제공할 수 있다.

---

### 4. 프로젝트 적용
본 프로젝트에서는 Socket.IO를 활용하여 WebSocket 기반 실시간 채팅 기능을 구현하였다.  

사용자가 채팅방에 입장하면 서버와 연결이 유지되고, 메시지를 전송할 경우 서버를 통해 같은 채팅방에 있는 사용자들에게 즉시 전달되도록 설계하였다.  
이를 통해 다수의 사용자가 동시에 참여하는 환경에서도 지연 없이 메시지를 주고받을 수 있도록 구현하였다.
### Database
- MongoDB: 유연한 스키마 기반 데이터 모델링 및 컬렉션 설계

커뮤니티 및 실시간 채팅 기능 구현을 위해 MongoDB를 사용하여 데이터 구조를 설계했습니다.

- chat / messages: 실시간 채팅 메시지 저장
- chatroom: 채팅방 단위 관리
- post / comment: 게시글 및 댓글 기능 분리
- user / people: 사용자 및 참여자 정보 관리
- invitation: 협업 초대 기능 지원
- sessions: 사용자 세션 관리

각 기능을 독립적인 컬렉션으로 분리하여 확장성과 유지보수성을 고려했으며,  
인덱스를 적용하여 조회 성능을 최적화했습니다.
